# Project Context

## Goal

I am developing a simulator of an Autonomous Surface Vehicle (ASV) using ROS 2 Jazzy and Gazebo Harmonic.

The simulation runs correctly in Gazebo, but I am trying to visualize all robot frames and sensors correctly in RViz2.

The objective is to have a clean TF tree generated by robot_state_publisher from an URDF/Xacro while Gazebo handles the physics and sensors.

---

## Current TF strategy

Desired TF tree:

map
└── odom
└── base_link
├── left_propeller
├── right_propeller
├── gps_link
├── sensors_link
└── sonar_link

robot_state_publisher should publish all robot internal transforms.

gazebo_odom.py should publish ONLY

odom -> base_link

Nothing else should publish those transforms.

---

## Current issue

robot_state_publisher publishes the URDF correctly.

However Gazebo's OdometryPublisher publishes

odom -> vessel_v2

instead of

odom -> base_link

while gazebo_odom.py publishes

odom -> base_link

This creates inconsistent TF trees.
